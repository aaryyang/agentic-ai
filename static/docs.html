Core API Endpoints

Agent Interaction

POST /agent/chat
Direct interaction with the AI agent for general conversation and task delegation 
{
  "message": "Qualify this lead: John Doe from Acme Corp",
  "user_id": "user_123",
  "context": {"source": "web_form"}
}
POST /agent/delegate
Delegate specific tasks to specialized agents (sales, operations, quotes, scheduler) 
{
  "agent_type": "sales",
  "task": "Update deal ABC123 to proposal stage",
  "context": {"deal_id": "ABC123"}
}
GET /agent/status
Check the current status of the AI agent system

External Platform Webhooks

POST /whatsapp/webhook
WhatsApp Business API webhook for processing incoming messages
POST /telegram/webhook
Telegram Bot API webhook for handling bot updates
POST /webchat/message
Web chat integration for embedded chat widgets 
{
  "session_id": "session_123",
  "message": "I need a quote for CRM integration",
  "user_info": {"name": "John", "email": "john@example.com"}
}

Workflow Automation

POST /workflows/create
Create automated workflow processes for business operations
POST /workflows/{workflow_id}/execute
Execute a specific workflow with provided parameters

Quick Start Guide

1. Basic Agent Chat

curl -X POST "http://localhost:8000/agent/chat" \
     -H "Content-Type: application/json" \
     -d '{
       "message": "Help me qualify a new lead",
       "user_id": "demo_user"
     }'

2. Delegate to Sales Agent

curl -X POST "http://localhost:8000/agent/delegate" \
     -H "Content-Type: application/json" \
     -d '{
       "agent_type": "sales",
       "task": "Analyze pipeline performance",
       "context": {"period": "last_month"}
     }'

3. Check System Status

curl -X GET "http://localhost:8000/agent/status"

Integration Examples

Python Integration

import requests

# Chat with the agent
response = requests.post("http://localhost:8000/agent/chat", json={
    "message": "Generate a quote for enterprise CRM",
    "user_id": "user_123",
    "context": {"budget": 50000}
})

print(response.json())

JavaScript Integration

const response = await fetch('/agent/chat', {
    method: 'POST',
    headers: {'Content-Type': 'application/json'},
    body: JSON.stringify({
        message: 'Schedule a follow-up meeting',
        user_id: 'user_123'
    })
});

const data = await response.json();
console.log(data);

Authentication

Currently, the AI Agent CRM System operates in development mode without authentication. For production deployment, implement:

  • API Keys: Bearer token authentication for API access
  • Rate Limiting: Request throttling to prevent abuse
  • CORS: Cross-origin resource sharing configuration
  • Input Validation: Comprehensive request validation

Error Handling

The API returns structured error responses with appropriate HTTP status codes:

Common HTTP Status Codes

  • 200 OK: Successful request
  • 400 Bad Request: Invalid request parameters
  • 404 Not Found: Endpoint not found
  • 500 Internal Server Error: Server-side error

Error Response Format

{
  "error": "Invalid request",
  "message": "Missing required field: message",
  "status_code": 400
}

Rate Limits

Current rate limiting (development mode):

  • Agent Chat: No limits (development only)
  • Webhooks: No limits (development only)
  • Workflows: No limits (development only)

Note: Production deployment should implement appropriate rate limiting based on your use case.
